## Mid-session pivot

Started the session planning to sync the user's YouTube Notion DB (`338110…`) into 2nd-brain. First test sync landed 100 rows — every single `raw_source` was exactly **2000 chars**, ending mid-sentence. That's Notion's hard API cap on `rich_text` property reads.

Looked upstream: `unsubject/YouTubeArchive/sync_to_notion.py` literally does `transcript[:2000]` on the write side. Notion has always been a lossy downstream of `videos.db`, never the source of truth. So pulling transcripts from Notion was structurally never going to work — the data isn't there to read.

Architecture corrected mid-session: 2nd-brain pulls from `videos.db` via a push from YouTubeArchive, not from Notion. The Notion DB stays the user's editorial UI but is no longer a 2nd-brain data source for YouTube. The 100 truncated rows were deleted; the `DATABASE_CONFIG` entry for the YouTube DB was removed from `notion-api.ts`.
